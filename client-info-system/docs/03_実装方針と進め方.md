# 利用者情報整理システム 実装方針と進め方

## 1. 技術選定（確定）

**基盤は Notion** で進めます（写真添付・編集のしやすさ、規模：利用者約100名・従業員約40名に対応）。

- 具体的なデータベース構成・テンプレート・権限 → **04_Notion設計.md**
- テキストを貼ると項目に振り分けるAI → **05_AIテキスト振り分け設計.md**

以下は参考として残しています。

---

### 参考：他の選択肢（今回は不採用）

### オプションB: Webアプリ＋DBを自作する

| 構成例 | メリット | デメリット |
|--------|----------|------------|
| **Next.js + Supabase（PostgreSQL）** | 認証・DB・ストレージが一通り揃う。無料枠で小規模なら運用可能。 | 初めてだと学習コストあり。 |
| **Next.js + Firebase** | 認証・Firestore・Storageが統合。モバイル対応も考えやすい。 | 複雑な検索・集計はFirestoreの制約を受ける。 |
| **React / Vue + バックエンド（Node等） + DB** | 自由度が高い。 | 設計・開発・ホスティングの手間が増える。 |

**おすすめ**: 長期的に拡張・AI連携を考えるなら、**Next.js + Supabase**で「利用者一覧→詳細→支援パターン」まで作る形がバランス良いです。

---

### オプションC: 既存の福祉系サービスを利用する

福祉・障がい者支援向けの業務支援ツール（利用者情報・支援記録を扱うもの）を導入する方法もあります。  
「自前でカスタマイズしたい」「既存のLINE運用から移行したい」という要件と照らして、自前開発と比較検討するとよいです。

## 2. 推奨する進め方（フェーズ案）※Notion・AI初期導入

### フェーズ0: 準備（1週間程度）

- [x] 要件・規模・必須項目・Notion採用・AI初期導入の確定（済）
- [ ] Notionワークスペースの準備（チームプラン推奨・約40名招待）
- [ ] **04_Notion設計.md** に沿って「利用者データベース」とテンプレートを作成
- [ ] AI入力ヘルパーの形態を決定（GASでWebアプリ / 手動コピペ用の出力のみ など）

### フェーズ1: Notionの構築とAI振り分けの並行（2〜3週間）

**目標**: 利用者情報をNotionで一覧・詳細まで見られるようにし、テキスト貼り付け→項目振り分けを運用開始時から使えるようにする。

- Notion上で利用者データベースのプロパティ・詳細ページのテンプレートを完成させる（全必須項目を配置）
- 支援の流れは「利用者ページの子ページ」で1パターン1ページのテンプレートを用意
- **05_AIテキスト振り分け設計.md** に基づき、AI入力ヘルパーを実装（まずは振り分け結果を表示→Notionにコピペで反映で可）
- サンプルで2〜3人分を手入力＋2〜3人分をLINE/紙のテキストでAI振り分けして入力し、項目の過不足を確認

### フェーズ2: データ投入と運用開始（2〜4週間）

**目標**: LINE 20〜30名分の移行、紙ベースで100名分入力できる体制にする。

- LINEの既存データをテキストで貼り、AI振り分けでNotionに反映（必要なら手修正）
- 紙の資料を入力する担当者に、テンプレートの使い方とAI入力ヘルパーの手順を共有
- 権限の確認（管理者・一般の閲覧/編集）

### フェーズ3: 運用の安定化（以降）

- バックアップ（Notionエクスポートの定期実施）
- 必要に応じてNotion APIで「振り分け結果をワンクリックでNotionに反映」を追加

---

## 3. 次のアクション

1. **Notionの実際の作成**: 04_Notion設計.md を参照し、利用者データベースとテンプレートをNotion上で作成する。
2. **AI入力ヘルパー**: 05_AIテキスト振り分け設計.md の「実装の選択肢」から、GASでWebアプリにするか・まずは手動コピペ用のみにするかを決め、プロンプトと出力形式を確定したうえで実装に進む。
3. **写真・オフライン**: 写真は1利用者あたり無制限。オフラインは想定せず、Notion上でオンライン参照が前提（要件定義書「7. 確定済みの方針」に記載済み）。
