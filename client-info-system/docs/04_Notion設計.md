# 利用者情報整理システム Notion設計

## 1. 前提

- **基盤**: Notion（写真添付・編集のしやすさを優先）
- **規模**: 利用者 約100名、従業員 約40名
- **必須項目**: 要件定義書「3. 機能要件」の項目をすべてNotion上に用意する

Notionは**チーム・有料プラン**を想定（メンバー約40名の招待、ストレージ、ブロック数制限のため）。  
無料プランではゲスト含め共有に制限があるため、運用開始前にプラン確認を推奨します。

---

## 2. ワークスペース構成案（確定方針）

**方針**: 画面遷移を極力減らし、情報リテラシーが高くない従業員でも迷わないようにする。

### 2.1 全体の流れ

```
[Webアプリ] 利用者情報一覧（氏名・よみがな・性別・移動 など）
       │
       │  行をクリック
       ▼
[Notion] その利用者さんの「1ページ」を開く（全情報がこの1ページ内）
```

- **利用者情報一覧**: **Webアプリ上で作成**する。一覧画面で氏名・よみがな・性別・移動などを表示し、各行から**その利用者のNotionページへリンク**で紐づける。
- **Notion**: **1利用者につき1ページ**。その1ページの中に**基本情報・住所・移動・意思疎通・危険・NG・食事・排泄・支援の情報・支援の流れ（パターン別）**を**すべて**入れる。子ページや別データベースにはせず、**同じページ内のセクション**として並べる。

これにより「一覧で利用者を選ぶ → 1ページ開く」の**2ステップ**で完結し、画面遷移を最小にできます。

### 2.2 Notion側の構成（シンプル案）

- 各利用者用のNotionページを1人1つ用意する（データベースのレコードとして作るか、通常ページを並べるかは運用で選択可）。
- Webアプリの一覧では、各利用者に対応する**NotionページのURL**を保持し、「Notionで開く」でその1ページに飛ばす。
- その1ページの中身は **4. 利用者「詳細ページ」のレイアウト** に従い、**4.9 支援の流れ**は子ページではなく**同じページ内の見出し**で複数パターンを並べる（後述）。

---

## 3. 情報一覧（Webアプリ）の構成

**一覧画面のタイトルは「情報一覧」**とし、**利用者情報**と**施設情報**の両方を含めます。  
**ひらがな（よみがな・読み仮名）で検索**できるようにし、利用者・施設を絞り込みます。

### 3.1 利用者情報の一覧に表示する項目

| 表示項目 | 説明 |
|----------|------|
| **氏名** | 利用者氏名 |
| **よみがな** | 検索・ソート用（ひらがな検索の対象） |
| **性別** | 男性 / 女性 / その他 |
| **連絡先（電話番号）** | 連絡先の電話番号 |
| **住所（自宅）** | 利用者詳細ページと同じ自宅住所。Googleマップへのリンクを付ける |
| **詳細を開く** | その利用者のNotionページ（1ページに全情報）へのリンク |

※一覧には顔写真・移動・金銭管理は出さない（画面をシンプルにし、詳細は1ページで確認する想定）。

### 3.2 施設情報の一覧に表示する項目

| 表示項目 | 説明 |
|----------|------|
| **施設名** | 施設名 |
| **読み仮名** | 検索用（ひらがな検索の対象） |
| **住所** | 住所。Googleマップへのリンクを付ける |
| **連絡先** | 施設の連絡先（電話番号など） |

### 3.3 ひらがな検索

- 検索欄に入力したひらがな（よみがな・読み仮名の一部）で、利用者一覧と施設一覧の両方を絞り込む。
- 例：「やまだ」「さとう」「ほうかご」「つうしょ」などで該当行だけ表示する。

一覧用のデータは、NotionデータベースのプロパティからAPIで取得するか、Webアプリ用の別データソースで持つかは実装時に選択します。

## 4. 利用者「詳細ページ」のレイアウト（テンプレート）

データベースのレコードを開いたときのページ＝**利用者1人分の詳細**です。  
以下の見出し（トグルまたは見出しブロック）で区切り、必須項目をすべて配置します。

### 4.1 基本情報

| 表示名 | 内容 | Notionでの入力形式 |
|--------|------|---------------------|
| 顔写真・写真 | 枚数制限なし | プロパティ「顔写真」で一覧用の代表1枚。ページ内には必要に応じて何枚でも画像ブロックで追加 |
| 性別 | 選択 | プロパティで管理（上記） |
| よみがな | 検索用 | プロパティで管理 |

### 4.2 住所

各「場所」を「見出し + テキスト + リンク」でまとめます。GoogleマップはURLをリンクとして貼れます。

| 項目 | 内容 |
|------|------|
| 自宅 | 住所テキスト ＋ Googleマップリンク |
| 放課後等デイサービス（あれば） | 同上 |
| 通所（あれば） | 同上 |
| 学校（あれば） | 同上 |
| その他よく行く場所（あれば） | 箇条書きまたは表で「場所名・住所・マップリンク」 |

**テンプレート例（コピペ用）:**

```
## 住所

### 自宅
（住所）
（Googleマップのリンク）

### 放課後等デイサービス
（あれば記入）

### 通所
（あれば記入）

### 学校
（あれば記入）

### その他よく行く場所
（あれば記入）
```

### 4.3 移動・意思疎通

| 項目 | 内容 |
|------|------|
| 移動 | プロパティ「移動」で選択済み。補足があればここに記述。 |
| 意思疎通 | どのような形でできるか、または「意思疎通不可」など（テキストブロック） |

### 4.4 危険・NG行為（重要）

| 項目 | 内容 |
|------|------|
| 危険な行為・命に関わる行為 | テキスト |
| ヘルパーがやってはいけないこと（NG行為） | テキスト |

### 4.5 食事

| 項目 | 内容 |
|------|------|
| 食事 | 自立 / 介助 ＋ 注意事項（テキスト） |

### 4.6 排泄

| 項目 | 内容 |
|------|------|
| 排泄 | 自立 / 介助 ＋ 注意事項（テキスト） |
| リハビリパンツ | 着用あり / なし |

### 4.7 その他（障害・病気・服薬）

| 項目 | 内容 |
|------|------|
| 障害・病気の情報 | テキスト |
| 服薬情報 | テキスト |

### 4.8 支援についての情報

| 項目 | 内容 |
|------|------|
| 主な支援の注意点・ポイント | テキスト |
| 金銭管理 | プロパティで「ヘルパー管理/本人管理」。詳細は以下。 |
| 交通費の支払い | 利用者側：現金/IC/その他。ヘルパー側：同様。 |
| ご家族からの要望（あれば） | テキスト |
| 好きなこと・喜ぶこと（あれば） | テキスト |

### 4.9 支援の流れ（パターン別）※同じ1ページ内に配置・時系列で丁寧に

**支援の流れはこのページでいちばん重要な部分**として、**時系列のステップ（①→②→③…）**で書けるようにします。  
「スタート」「ゴール」「タイムスケジュール」を別項目にせず、**流れに沿って①、②、③…と順に書き、各ステップに「いつ・どこで・何をする・注意点」**をまとめて書きます。

- パターンが複数ある場合（例：自宅→放デイ、学校送迎）、それぞれを **### パターン名** で区切り、その下に ①→↓→②→↓→③… の形でステップを並べる。
- **写真・金銭・排泄・問題時の対処は、基本的に該当するステップの中に書く。** 時間・場所・やることとあわせて、そのステップで必要な注意を同じブロックにまとめる。写真は該当ステップの前後にNotionの画像ブロックで挿入する。
- ステップ数はパターンごとに増減してよい（③で終わりでも、⑦まであってもよい）。

**1ページ内でのテンプレート例（時系列ステップ形式）:**

```
## 支援の流れ

### パターン1：自宅→放デイ→自宅

① （例：〇時 自宅到着・挨拶。スタート場所の住所・マップリンク。鍵の扱いなど）
↓
② （例：一緒に最寄り駅まで。電車は〇〇線。交通費の支払い方。注意点）
↓
③ （例：放デイ到着。スタッフへの引き継ぎ。排泄は〇〇で対応）
↓
④ （例：帰り 迎え〜自宅まで。ゴール。問題時の連絡先）

（写真・金銭・排泄・問題時の対処は、基本的に該当するステップの中に書く）

### パターン2：学校送迎

① スタート：自宅（住所・マップリンク）
↓
② （やること・注意点・時間）
↓
③ ゴール：学校 など
```

パターンが増えても、**同じ利用者ページ内で「パターン名＋①→②→③…」を増やすだけ**にし、別ページへ飛ばない運用にします。

## 5. Notion「1利用者1ページ」のテンプレート設定

1. 利用者用のNotionページを1つ用意し、**テンプレート**として保存するか、新規作成時にコピーする元にする。
2. テンプレートの**本文**に、4.2〜4.9 の**すべて**の見出しとプレースホルダーを入れておく（支援の流れは同じページ内に複数パターン分の見出しを並べる形式）。
3. 新規利用者を追加するときは、このテンプレートをコピーして1ページ作り、タイトルを利用者名にする。Webアプリの一覧には、このページのURLを登録する。

これで、100名分の入力時も構成がぶれず、AI振り分けで埋める項目とも対応できます。

---

## 6. 権限・共有（約40名）

- **ワークスペース**: 事業所用のワークスペースを1つ用意し、従業員を**メンバー**として招待（有料プランで人数制限を確認）。
- **利用者情報の親ページ**: メンバー全員に「編集」または「閲覧」を付与。編集は管理者のみにしたい場合は、一般メンバーは「閲覧」、管理者だけ「編集」に分ける。
- **データベース**: 親ページの権限を継承するため、個別設定は基本的に不要。サブページも同様。

退職者には、Notionのメンバー削除でアクセスを無くせます。

---

## 7. 運用のコツ

- **検索**: Notionの検索（Ctrl/Cmd + K）で利用者名・よみがな・本文のキーワードを検索可能。
- **オフライン**: 想定しない。利用はすべてNotion上でのオンライン参照が前提。
- **バックアップ**: Notionの「エクスポート」でワークスペースまたはページ単位のHTML/PDF/Markdownを定期取得すると安心です。

---

## 8. AI振り分けとの対応

「テキストを貼ると項目に振り分ける」機能では、次の対応関係でAIが出力し、Notionに反映します。

- **利用者ページ**: セクション見出し（## 住所、## 意思疎通 など）と、その下のブロック（テキスト・箇条書き）に対応させる。
- **支援パターン**: 同じ利用者ページ内の見出し（### パターン1：〇〇、### パターン2：〇〇）の下に、「スタート・ゴール」「タイムスケジュール」などの内容を振り分ける。

具体的な項目名とAI出力形式は **05_AIテキスト振り分け設計.md** を参照してください。
