# 利用者情報整理システム 要件定義書（たたき台）

## 1. 背景・目的

### 1.1 現状の課題
- **情報の抜け漏れ**: LINEノート・グループでのテキスト共有では、誰が何を更新したか追いにくく、古い情報が残りやすい
- **更新のしづらさ**: 時系列のチャット形式のため、「この利用者さんの最新の注意点だけ知りたい」といった参照が難しい
- **入力形態**: テキスト・紙資料が多く、デジタル化・構造化のニーズがある

### 1.2 システムの目的
- 利用者様情報を**一箇所に集約**し、従業員が**必要な情報をすぐ参照**できるようにする
- **支援の流れ（パターン別・時系列）**を整理し、新人・代替ヘルパーでも安心して支援できるようにする
- **AIを最初から活用**し、テキストを貼り付けると項目に振り分けて入力しやすくする

### 1.3 事業内容
- **移動支援**
- **居宅支援**

---

## 2. 想定ユーザー

| 役割 | 主な利用シーン |
|------|----------------|
| 管理者 | 利用者登録・編集、権限管理、情報の一括確認 |
| 正職員・ヘルパー | 支援前の情報確認、支援中の参照、支援後のメモ追記 |
| 新人・代替 | 初めての利用者支援時の流れ・注意点の確認 |

---

## 3. 機能要件（大分類）

### 3.1 利用者マスタ（基本情報・共有情報）

従業員に共有する「利用者様の人物像・支援の前提」を管理する。

| 項目 | 内容 | 備考 |
|------|------|------|
| 顔写真・写真 | 枚数制限なし | 代表写真は一覧用に便利。本人確認・安心感のため。その他も必要に応じて何枚でも |
| 基本情報 | 性別 | |
| 住所関連 | 自宅、放デイ、通所、学校、その他よく行く場所 | 各住所にGoogleマップリンクを保持 |
| 移動 | 歩行自立／補助／車椅子 | 選択式想定 |
| 意思疎通 | 方法の説明、または「意思疎通不可」 | 自由記述＋選択 |
| 危険・NG | 危険な行為・命に関わる行為、ヘルパーがやってはいけないこと | 自由記述、重要度高 |
| 食事 | 自立／介助、注意事項 | |
| 排泄 | 自立／介助、注意事項、リハビリパンツの有無 | |
| その他 | 障害・病気、服薬情報 | |

### 3.2 支援についての情報

| 項目 | 内容 |
|------|------|
| 主な支援の注意点・ポイント | 自由記述 |
| 金銭管理 | ヘルパー管理／本人管理、電車・バスの支払い方法（現金／IC／その他）、利用者側・ヘルパー側それぞれの支払い方法 |
| ご家族からの要望 | あれば |
| 好きなこと・喜ぶこと | あれば |

### 3.3 支援の流れ（パターン別）

**「どのパターンで支援するか」ごとに、時系列で流れを整理する。**

| 項目 | 内容 |
|------|------|
| パターン名 | 例：「自宅→放デイ→自宅」「学校送迎」「外出支援（〇〇方面）」 |
| 支援スタート場所 | 住所＋マップリンク |
| 支援終了場所 | 同上 |
| 時系列の流れ | 時間帯、やること、注意点を時系列で記載 |
| 写真 | 必要に応じて挿入（乗り場、建物入口など） |
| 金銭まわり注意点 | そのパターン特有の支払い・精算の注意 |
| 排泄まわり注意点 | そのパターン特有のトイレ・オムツ等 |
| タイムスケジュール | 想定される時間配分 |
| 外出支援時 | 行き先のおすすめ、よく行く場所 |
| 問題発生時の対処 | よくあるトラブルと対処方法 |

## 4. 非機能要件（検討項目）

| 項目 | 検討内容 |
|------|----------|
| アクセス制御 | 自事業所の従業員のみ、またはロール（管理者／一般）で制限するか |
| 端末 | 主にスマホか、PCも使うか（画面設計に影響） |
| オフライン | 想定外（Notion上でオンライン参照が前提） |
| 個人情報 | 氏名・住所・写真の取り扱い、バックアップ、退職者アクセス削除 |
| AI活用 | 紙・テキストの要約入力、キーワード検索の自然言語化、注意点のハイライトなど |

---

## 5. 確定事項（2025年2月時点）

| 項目 | 内容 |
|------|------|
| **基盤** | **Notion**（写真添付・編集のしやすさを優先） |
| **規模** | 利用者 約100名、従業員 約40名 |
| **既存データ** | LINE：20〜30名分を移行。紙：100名分入力できる構造で準備 |
| **必須項目** | 本要件定義の「3. 機能要件」に挙げた項目はすべて必須 |
| **AI** | テキストを貼ると項目に振り分ける機能を**最初から**利用可能にする |

---

## 6. 用語整理

- **利用者様**: サービスを受ける方（障害者・高齢者等）
- **ヘルパー**: 移動支援・居宅支援を実施する従業員
- **パターン**: 支援の形態・ルートの類型（例：送迎A、外出B）

---

## 7. 確定済みの方針（写真・オフライン）

- **写真**: 1利用者あたりの枚数制限は設けない（無制限）。代表写真は一覧用にあると便利だが、それ以外も必要に応じて何枚でも添付可能。
- **オフライン**: 利用は想定しない。基本的にNotion上でオンライン参照する前提。

※「何枚まで」を聞いていた理由は、ストレージや画面設計の目安を取るためでした。Notionはワークスペース単位のストレージなので、利用者ごとの上限を設ける必要はなく、無制限で運用します。
