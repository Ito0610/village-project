# AIテキスト振り分け 設計

## 1. 目的

- 紙の資料やLINEのテキストなど**非構造化テキスト**を貼り付けると、AIが**利用者情報の項目ごとに振り分け**し、Notionに反映しやすくする。
- **運用開始時から**使えるようにする。

Notion単体には「自由テキストを項目に自動振り分け」する機能がないため、**外部の「AI入力ヘルパー」**を用意します。

---

## 2. 全体の流れ

```
[ユーザー] テキストを貼り付け
      ↓
[AI入力ヘルパー] テキストを送信
      ↓
[AI（OpenAI / Claude 等）] 項目ごとに要約・振り分け → 構造化データ（JSON）を返す
      ↓
[AI入力ヘルパー] 結果を表示
      ↓
  (A) ユーザーがNotionにコピペで反映
  または
  (B) Notion API で指定ページを更新（半自動）
```

- **まずは (A)** で、振り分け結果を「Notionに貼りやすい形式」で表示し、ユーザーがコピーして該当セクションに貼る運用が簡単です。
- **(B)** は Notion API と Integration の設定が必要ですが、ワンクリックでページを更新できます。後から導入可能です。

---

## 3. 振り分け先の項目一覧（AIの出力仕様）

AIには「次のキーに沿ってJSONで返す」と指示します。キーはNotionのセクション名・プロパティ名と対応させます。

### 3.1 利用者マスタ用（1利用者1ページに対応）

| キー（英語） | 日本語項目名 | 説明 |
|--------------|--------------|------|
| name | 利用者氏名 | テキストから読み取れる場合のみ |
| name_kana | よみがな | あれば |
| gender | 性別 | 男性/女性/その他/不明 |
| address_home | 自宅住所 | 住所＋GoogleマップURLは別途ユーザーが追記可 |
| address_after_school | 放デイ（あれば） | 同上 |
| address_tsusho | 通所（あれば） | 同上 |
| address_school | 学校（あれば） | 同上 |
| address_other | その他よく行く場所 | 複数は改行または箇条書きで |
| mobility | 移動 | 歩行自立/補助/車椅子/不明 |
| communication | 意思疎通 | 方法の説明、または「不可」など |
| dangerous_behaviors | 危険な行為・命に関わる行為 | そのまま要約または抽出 |
| ng_behaviors | ヘルパーがやってはいけないこと | 同上 |
| eating | 食事 | 自立/介助＋注意事項 |
| toileting | 排泄 | 自立/介助＋注意事項 |
| toileting_rehab_pants | リハビリパンツ | あり/なし/不明 |
| disability_illness | 障害・病気の情報 | テキスト |
| medication | 服薬情報 | テキスト |
| support_points | 主な支援の注意点・ポイント | テキスト |
| money_management | 金銭管理 | ヘルパー管理/本人管理/不明 |
| transport_payment_client | 利用者側の交通費 | 現金/IC/その他、補足 |
| transport_payment_helper | ヘルパー側の交通費 | 同上 |
| family_requests | ご家族からの要望 | テキスト |
| likes_and_rejoice | 好きなこと・喜ぶこと | テキスト |
| memo | その他（上記に当てはまらないメモ） | テキスト |

### 3.2 支援パターン用（1パターン1子ページに対応）

テキストが「支援の流れ」の説明のとき用。

| キー（英語） | 日本語項目名 |
|--------------|--------------|
| pattern_name | パターン名（例：自宅→放デイ→自宅） |
| start_place | 支援スタート場所 |
| end_place | 支援終了場所 |
| timeline_schedule | タイムスケジュール |
| flow_steps | 支援の流れ（時系列のやること・注意点） |
| money_notes | 金銭まわり注意点 |
| toileting_notes | 排泄まわり注意点 |
| recommended_destinations | 行き先のおすすめ・よく行く場所 |
| trouble_shooting | 問題発生時の対処 |

## 5. 実装の選択肢

### 5.1 Google Apps Script（GAS）で作る

- 既存でGASを利用されているため、同じ環境で「AI入力ヘルパー」をWebアプリとして公開できる。
- **流れ**: スプレッドシートまたはサイドパネルに「テキストを貼る欄」と「振り分け実行」ボタン → GASでOpenAI API（またはVertex AI）を呼ぶ → 返ってきたJSONを「Notionに貼りやすい形」に整形して表示（別シートやダイアログ）。
- **Notion API連携**: GASからNotion APIを呼び、指定したページIDの子ブロックを追加することも可能。その場合はNotionのIntegrationトークンと、対象ページの共有設定が必要。

**必要なもの**: OpenAI APIキー（またはGoogleのVertex AI等）、（Bをやる場合）Notion Integration トークン、対象データベース/ページのID。

### 5.2 小型Webアプリ（Next.js等）で作る

- 入力フォーム → AI API呼び出し → 結果表示（コピーボタン付き）。オプションでNotion APIでページ更新。
- デプロイ先（Vercel等）とAPIキーの管理が必要。

### 5.3 まずは「手動コピペ」で運用（Aのみ）

- AI部分だけGASや簡易スクリプトで「テキスト → JSON → 見出し付きテキスト」に変換し、出力をユーザーがNotionの該当セクションに貼る。
- Notion APIは使わないので、Notion側の設定は不要。最も早く使い始められる。

---

## 6. 次のステップ（実装時）

1. **AI入力ヘルパーの形態を決定**: GASでWebアプリ / 小型Webアプリ / まずは手動コピペ用の出力だけ。
2. **利用者用プロンプトを確定**: 上記プロンプトを実際のテキストで試し、項目の抜け・誤振り分けを減らすよう文言を調整。
3. **支援パターン用プロンプトを用意**: 同様に「支援の流れ」用のJSONスキーマとプロンプトを作成。
4. **（Bをやる場合）Notion Integration の準備**: NotionでIntegrationを作成し、利用者データベースおよび対象ページで「接続」し、ページIDを取得。GASまたはWebアプリからAPIで更新する処理を実装。

必要であれば、GASでの「テキスト入力 → OpenAI呼び出し → 見出し付きテキスト出力」のサンプルコードも別ファイルで用意できます。
