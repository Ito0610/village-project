# AI精度テスト手順（1利用者1ページ形式で出力）

テキストや利用者情報を整理した画像をAIに渡し、**1利用者1ページと同じ形式のマークダウン**で出力してもらい、Notionに貼り付けて振り分け精度を試すための手順です。

---

## 1. やりたいこと

- **入力**: 利用者さんの情報が書かれたテキスト、または情報を整理した画像（紙の資料の写真など）
- **出力**: 当システムの「1利用者1ページ」と同じ**形式のマークダウン**（見出し・項目の並びを揃えたもの）。**Notionにそのまま貼り付ける**想定
- **目的**: AIがどこまで正確に項目を振り分けられるか試し、プロンプトや運用を改善する

---

## 2. 対応の流れ（どうやるか）

### パターンA: チャットでAIに依頼する（手軽に試す）

1. **プロンプトをコピーする**  
   下記「3. サンプルプロンプト」の文をそのままコピーします。
2. **AIのチャット（Cursor / ChatGPT / Claude 等）を開く**
3. **1通目**: プロンプトを貼り付けて送信する。「この形式で出力して」と指示が書いてあります。
4. **2通目**: 利用者情報のテキスト（または画像）を貼り付けて送信する。  
   - テキストのみ: そのまま貼る  
   - 画像: 画像対応のAI（GPT-4V、Claude など）なら画像を添付して「この画像の情報を、先ほどの形式でHTMLにまとめてください」と送る
5. **出力されたマークダウン**をコピーし、Notionの利用者ページに貼り付ける。項目の抜け・誤振り分けがないか確認する。

### パターンB: プロンプトと入力を1回で送る

- プロンプト＋「以下が利用者情報です」＋テキスト（または画像）を**1回のメッセージ**で送る方法です。  
- 長文になるので、テキストが短いときや、画像1枚だけのときに向いています。

### パターンC: APIで自動化する（将来）

- GASやWebアプリから OpenAI API / Claude API を呼び、  
  「入力テキスト → プロンプト＋テキストを送信 → 返ってきたマークダウンをNotionに反映」  
  という流れを自動化できます。  
- まずはパターンAで手動テストし、精度が十分だと判断できたらAPI化を検討するとよいです。

---

## 3. サンプルプロンプト（コピー用）

以下をAIに送る前に貼り付けて使ってください。**利用者情報のテキストや画像は、この後に続けて送る**か、同じメッセージの末尾に追加してください。  
プロンプト本文は `06_AI用プロンプト_コピー用.txt` にあります（**マークダウン形式でNotionに貼り付け**用）。

- **出力**: マークダウン（# ## ### の見出し、リスト、リンクは [テキスト](URL) ）。Notionにそのまま貼り付けて使う。
- **テキストで渡す場合**: 「【これから渡すもの】」の下に、利用者情報のテキストをそのまま貼り付けて送信。
- **画像で渡す場合**: プロンプトを送ったあと、画像を添付して「この画像の内容を、先ほどの形式でマークダウンにまとめてください」と送る。

---

## 4. 出力形式の詳細（マークダウン・Notion用）

AIの出力がぶれないように、次の見出しと順序でマークダウンを出してもらいます。Notionは `#` `##` `###` を見出しに変換します。

| セクション | マークダウン | 補足 |
|------------|------------------|------|
| 氏名 | # 氏名 | ページタイトルとして先頭に |
| 基本情報 | ## 基本情報 | 性別・よみがな |
| 住所 | ## 住所 → ### 自宅 など | 住所とリンクは [Googleマップ](URL) |
| 移動・意思疎通 | ## 移動・意思疎通 | 移動：歩行自立/補助/車椅子。意思疎通の方法 |
| 危険・NG | ## 危険・NG行為（重要） | 危険な行為とNG行為を分けて記述 |
| 食事 | ## 食事 | 自立/介助＋注意事項 |
| 排泄 | ## 排泄 | 同上＋リハビリパンツ |
| その他 | ## その他（障害・病気・服薬） | 障害・病気、服薬 |
| 支援について | ## 支援についての情報 | 注意点・金銭・交通費・家族要望・好きなこと |
| 支援の流れ | ## 支援の流れ → ### パターン名 | ①→②→③… のステップ形式。注意点は該当ステップ内に |

参照用のページ構造は `mock/1利用者1ページ.html` と同じ並びです。マークダウンにするとNotionに貼り付けたときに見出しが階層化されます。

---

## 5. テスト用サンプル入力（テキスト例）

以下は「LINEやメモからコピーしたような」想定のテキストです。このテキストをAIに渡して、HTMLが期待どおりに振り分けられるか試せます。

```
【利用者】佐藤花子（さとうはなこ）女性
【自宅】東京都○○区○○ 2-3-4（マップはあとで入れる）
【移動】車椅子。段差は介助で。
【意思疎通】簡単な言葉とジェスチャー。難しいときは家族に電話確認可。
【注意】急に立ち上がろうとすることがある。転倒に注意。無理に止めない。
【食事】介助。むせやすいのでとろみ付き。ゆっくり。
【排泄】オムツ。リハビリパンツ。トイレ誘導は2時間おき程度。
【服薬】朝・夕で2種類。名前は〇〇と〇〇。
【金銭】ヘルパー管理。電車は利用者IC、ヘルパーは現金。
【家族から】水分多めに。暑い日は無理しないで。
【好きなこと】お菓子、音楽を聴く。
【支援の流れ・送迎】自宅から放デイまで。8時半に自宅。一緒にバスで〇〇行き。降りてから5分。放デイ到着でスタッフに引き継ぎ。帰りは16時迎え、同じ経路で自宅まで。
```

このサンプルをプロンプトの「【これから渡すもの】」の下に貼り、AIにマークダウンを出力させ、  
Notionに貼り付けて「危険・NG」に転倒注意が入っているか、「支援の流れ」が①→②→③になっているか、などを確認するとよいです。

---

## 6. 精度の確認ポイント

- 氏名・よみがな・性別が正しいセクションに入っているか
- 住所・移動・意思疎通・食事・排泄がそれぞれの見出しの下にあるか
- 「急に立ち上がる」「転倒注意」などが**危険・NG**に入っているか
- 金銭・交通費・家族要望・好きなことが**支援について**に入っているか
- **支援の流れ**が時系列のステップ（①→②→③）に分解されているか
- 情報がどこにも振られず抜けていないか

ここをチェックしつつ、抜けや誤りが多ければ「〇〇は危険・NGに入れて」などプロンプトを少しずつ修正して再度試すと、精度を上げていけます。

---

## 7. 画像を渡す場合の注意

- **画像対応のAI**（GPT-4V、Claude 3 など）であれば、紙の資料の写真や手書きメモの画像を添付できます。
- 送り方の例: 「先ほどの1利用者1ページ形式で、この画像に写っている利用者情報をマークダウンにまとめてください。Notionに貼り付けます」
- 手書きは読み間違いがある可能性があるので、出力後に目視で確認してください。

---

以上で、テキストや画像を渡して「1利用者1ページ形式のマークダウン」を出してもらい、**Notionに貼り付ける**流れと、サンプルプロンプト・テスト用テキストをそろえています。  
精度を試した結果や「ここを直したい」という点があれば、プロンプト文やこの手順を一緒に調整できます。
