# コピペシート → 各利用者シート 転送マクロ

## 概要

シート「コピペシート」の2行目以降のデータのうち、**I列（請求）が「大田区」の行だけ**を、対応する利用者様シート（例：利用者名が「山田」なら「山田様」シート）の16〜35行目へ転記するExcel VBAマクロです。

## コピペシートの構成（2行目から）

| 列 | 内容 |
|----|------|
| A | サービス提供者名（スペース区切りで複数名可） |
| B | 利用者様名（「様」付きで入力。例: 藤井舞純様。この値がそのままシート名として参照されます） |
| C | 開始時間 |
| D | 終了時間 |
| E | 目的地 |
| F | 日付（YYYY/M/D） |
| G | 目的コード |
| H | 派遣人数 |
| I | 請求 |

## 動作ルール

- **I列が「大田区」**  
  → その行を、B列の値（例: 藤井舞純様）と同名のシートへ転記します。B列には既に「様」を含めて入力してください。
- **I列が「大田区」以外**  
  → 転記せず、コピペシート上でその行のA〜I列を**薄いグレー**で塗ります。
- **I列は「大田区」だが、該当する「〇〇〇〇様」シートがない／エラー**  
  → 転記せず、その行のA〜I列を**赤**で塗ってエラーと分かるようにします。
- **同一利用者で、同じ時間帯が重なっているレコード**  
  → 終了と開始がぴったりつながっているだけの場合は対象外。  
  それ以外の「時間帯の重なり」がある行のC〜D列を**黄色**で塗ります。

## 利用者シートへの転記先（16〜35行目）

| 列 | 内容 | コピペシートからの対応 |
|----|------|------------------------|
| A | 日（日付の「日」のみ。例：1） | F列の日付から日のみ |
| C〜E | 目的地（結合セル） | E列 |
| F | 目的コード | G列 |
| G, H | 開始時間・終了時間 | C列・D列 |
| J, K | 開始時間・終了時間（2回目） | C列・D列 |
| M | 派遣人数 | H列 |
| P | サービス提供者名 | A列を「派遣人数」ぶんだけ先頭から使用 |

### サービス提供者名の振り分け

- コピペシートのA列は「スペースで区切られた複数名」です（半角・全角スペース対応）。
- **派遣人数が1** → 先頭1人分の名前をP列に記載。
- **派遣人数が2** → 先頭2人分の名前をP列に記載。

例：A列が「市川　奥原(初)　荻原」で派遣人数が2 → P列には「市川　奥原(初)」のように2人分まで。

## 前提：レコード欄と21件目からの新シート

- 各利用者シートの**16行目〜35行目**がレコードを追加できる欄（**20件分**）です。
- **21件目**からは新しいシートを作成する前提です。
- 新シート作成時は、**元の「〇〇〇〇様」シートをそのまま複製**し、見た目・数式・書式は全く同じにします。
- **以下の列の16〜35行目だけ**は転記用のため、複製後に内容をクリアした状態にします。それ以外の部分は元シートと同じ内容のままです。
  - A列（日）、C〜E列（目的地）、F列（目的コード）、G・H列（開始・終了）、J・K列（開始・終了）、M列（派遣人数）、P列（サービス提供者名）
- シート名は「〇〇〇〇様(2)」「〇〇〇〇様(3)」… の順で作成します。

## 使い方

1. Excelで「コピペシート」に上記の形式でデータを入力（2行目から）。
2. 各利用者用のシートを「〇〇〇〇様」という名前で用意する。
3. VBAエディタで `CopyPasteToUserSheets.bas` を開き、マクロ **`CopyPasteToUserSheets`** を実行する。
4. 確認メッセージで「はい」を選ぶと転送が実行され、完了メッセージが表示される。

## エラー「インデックスが有効範囲にありません」が出る場合

マクロではエラー発生時に**処理段階・行番号・利用者名**などをダイアログで表示するようにしています。  
再度実行し、表示された「処理段階」と「コピペシートの行」の内容を控えてください。  
列構成が仕様と違う場合にこのエラーが出ることがあります。以下を確認してください。

- **コピペシート**: 1行目は見出しで、2行目からデータ。A〜I の 9 列を使っているか。
- **利用者シート**: 「〇〇〇〇様」は**ワークシート**（グラフシートではない）か。16〜35 行目に A, C〜E, F, G, H, J, K, M, P 列があるか。
