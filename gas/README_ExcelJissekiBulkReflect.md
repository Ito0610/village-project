# Excel実績 請求・目的コード・派遣人数 一括反映（GAS）

利用者様名が入ったセルを選択し、**請求**（9列右）・**目的コード**（7列右）・**派遣人数**（8列右）を一括で反映します。

---

## 仕様

### 前提
- **選択するセル** = 利用者様名が入っているセル（複数可）
- 同じ行の「1列左」= ヘルパー名、「4列右」= 目的地（参照）、「7列右」= 目的コード入力先、「8列右」= 派遣人数入力先、「9列右」= 請求入力先

### ◆請求（9列右のセル）

| 選択セルの背景色 | 処理 |
|------------------|------|
| `#fce5cd` | **スルー**（何も記入しない） |
| `#ff9900` または `#ffff00` | 「**居宅**」を入力（この行は目的コード・派遣人数は行わない） |
| 上記以外 | 「Excel実績自動化フラグ」**I列**（3行目以降）に同じ利用者様名があれば「**大田区**」を入力 |

「大田区」を入力した行のみ、以下を実行します。

### ◆目的コード（7列右のセル）

- **4列右**のセル（目的地）の値が「Excel実績自動化フラグ」**E列**（3行目以降）のいずれかと**部分一致** → 「**F**」
- それ以外 → 「**A**」

### ◆派遣人数（8列右のセル）

- 利用者様名が「Excel実績自動化フラグ」**A列**（3行目以降）のいずれかと**完全一致** かつ、**1列左**のヘルパー名が**2人以上**（スペース区切りでカウント）→ 「**2**」
- 上記以外（ヘルパー1人だけ、または利用者様名がリストにない）→ 「**1**」

### ◆スキップ条件

- **7列右**（目的コード）・**8列右**（派遣人数）・**9列右**（請求）のいずれかに既に値が入っている場合は、そのセル（行）は処理しません。既存の値を上書きしません。

---

## セットアップ（詳細手順）

「Excel自動化」メニューから一括反映を使う場合の、Apps Script エディタでの取り込み手順です。

### 前提

- 一括反映は **applyBulkReflect** という関数で動きます
- この関数は **ExcelJissekiBulkReflect.gs** に書かれています
- メニュー「【一括反映】目的コード・派遣人数・請求」は **ExcelJissekiFAInput.gs** の **addExcelAutomationMenu()** で追加されています
- そのため、**両方の .gs ファイル**を同じ Apps Script プロジェクトに置く必要があります

---

### ステップ1：Apps Script エディタを開く

1. 対象の **Google スプレッドシート**を開く
2. メニュー **拡張機能** → **Apps Script** をクリック
3. 新しいタブで「Apps Script エディタ」が開く（プロジェクト名は「無題のプロジェクト」など）

---

### ステップ2：一括反映用の新規 .gs ファイルを追加する

1. 左側の **ファイル一覧**（「ファイル」や「コード.gs」などが並んでいる部分）で、**＋**（プラス）ボタンをクリック
2. 表示されるメニューから **「スクリプト」** を選ぶ
3. 新しいファイルが追加され、名前が「無題のプロジェクト」や「コード」のような状態になる
4. その **ファイル名**（左の一覧でクリックして選択）の右側の **︙**（縦三点）をクリック → **「名前を変更」** を選ぶ
5. 名前を **ExcelJissekiBulkReflect** に変更して Enter（拡張子 .gs は自動で付きます）
6. エディタの **真ん中の編集エリア**をクリックし、**ExcelJissekiBulkReflect.gs** の内容を**すべて**コピーして貼り付ける  
   - コピー元：このリポジトリの **gas/ExcelJissekiBulkReflect.gs** を開き、全文選択（Ctrl+A / Cmd+A）→ コピー
7. メニュー **ファイル** → **保存**（または Ctrl+S / Cmd+S）で保存する

これで「一括反映」の処理がプロジェクトに追加された状態になります。

---

### ステップ3：「Excel自動化」メニュー用のファイルを追加する

「Excel自動化」メニュー（目的コード反映・2人付反映・請求フラグ・**一括反映**など）を表示するには、**ExcelJissekiFAInput.gs** も同じプロジェクトに必要です。

#### すでに「Excel自動化」メニューがある場合（ExcelJissekiFAInput.gs を既に取り込んでいる場合）

- そのファイルに、すでに「【一括反映】目的コード・派遣人数・請求」を追加する処理が入っています
- **ステップ2** で **ExcelJissekiBulkReflect.gs** を追加して保存すれば、メニューから一括反映を実行できるようになります
- ここで **ステップ4** へ進んでください

#### まだ「Excel自動化」メニューがない場合（ExcelJissekiFAInput.gs をまだ取り込んでいない場合）

1. 左のファイル一覧で再度 **＋** → **「スクリプト」** で新しいファイルを追加
2. ファイル名を **ExcelJissekiFAInput** に変更（ステップ2の 4～5 と同様）
3. このリポジトリの **gas/ExcelJissekiFAInput.gs** の内容を**すべて**コピーして貼り付け、**保存**
4. **onOpen** でメニューを出しているか確認する：
   - プロジェクト内の **コード.gs** や **Macro.gs** など、**onOpen** が書いてあるファイルを開く
   - 中身に次のような **onOpen** があるか確認する：
     ```javascript
     function onOpen() {
       addExcelAutomationMenu();
       // 他のメニューがあればここに addItem など
     }
     ```
   - **addExcelAutomationMenu();** が無い場合は、`onOpen` の `{` の直後に  
     `addExcelAutomationMenu();`  
     の行を 1 行追加して保存する

これで「Excel自動化」メニューに「【一括反映】目的コード・派遣人数・請求」が含まれた状態になります。

---

### ステップ4：スプレッドシートを再読み込みする

1. **スプレッドシートのタブ**に戻る
2. ブラウザで **更新（F5）** するか、一度スプレッドシートを閉じて開き直す
3. **onOpen** が実行され、メニューが再作成される
4. メニューバーに **「Excel自動化」** が表示され、その中に **「【一括反映】目的コード・派遣人数・請求」** があればセットアップ完了です

---

### 一括反映だけを別メニューで使う場合

「Excel自動化」は使わず、一括反映だけをメニューに出したい場合は次のようにします。

1. **ExcelJissekiBulkReflect.gs** だけを上記の手順で追加する
2. **onOpen** が書いてあるファイル（コード.gs など）を開き、次のように **addBulkReflectMenu();** を追加する：
   ```javascript
   function onOpen() {
     addBulkReflectMenu();  // これで「【一括反映】目的コード・派遣人数・請求」という名前のメニューが1つ表示される
   }
   ```
3. 保存してスプレッドシートを再読み込みする

## セットアップ（概要）

1. スプレッドシートの **拡張機能** → **Apps Script** を開く
2. **ExcelJissekiBulkReflect.gs** を新規ファイルとして追加する
3. メニューを出すため、次のいずれかを行う
   - **Excel自動化メニューを使っている場合**：`ExcelJissekiFAInput.gs` の `addExcelAutomationMenu()` に本スクリプトの項目が追加済みです。`onOpen()` で `addExcelAutomationMenu()` を呼んでいれば、**Excel自動化** → **【一括反映】目的コード・派遣人数・請求** が表示されます
   - **このスクリプトだけ使う場合**：プロジェクト内の `onOpen()` で `addBulkReflectMenu();` を呼び出す（別メニュー「【一括反映】目的コード・派遣人数・請求」が追加されます）
4. スプレッドシートを再読み込みする

---

## 使い方

1. **利用者様名**が入っているセル範囲を選択する（B列以降であること。1列左のヘルパー名・9列右までの列が必要）
2. メニュー **Excel自動化** → **【一括反映】目的コード・派遣人数・請求** を実行する
3. 請求・目的コード・派遣人数が一括で入力される（完了後、メッセージは表示されません）

## トラブルシューティング：「Excel自動化」は出るが一括反映が反映されない

### 原因1：ExcelJissekiBulkReflect.gs がプロジェクトにない（最も多い）

「Excel自動化」メニューは **ExcelJissekiFAInput.gs** だけで表示されますが、**【一括反映】目的コード・派遣人数・請求** を実行する処理（`applyBulkReflect`）は **ExcelJissekiBulkReflect.gs** に書かれています。

- **症状**：メニューに「【一括反映】目的コード・派遣人数・請求」はあるが、クリックすると「applyBulkReflect が見つかりません」などのエラーになる、または何も起きない
- **対処**：Apps Script エディタの左のファイル一覧に **ExcelJissekiBulkReflect**（または ExcelJissekiBulkReflect.gs）があるか確認する。無い場合は、[セットアップ（詳細手順）](#セットアップ詳細手順)の **ステップ2** のとおり、新規スクリプトを追加し、**gas/ExcelJissekiBulkReflect.gs** の内容をすべて貼り付けて保存する

### 原因2：メニューに「【一括反映】～」の項目が表示されていない

- **対処**：**ExcelJissekiFAInput.gs** を開き、`addExcelAutomationMenu` の中に次の 1 行があるか確認する  
  `.addItem('【一括反映】目的コード・派遣人数・請求', 'applyBulkReflect')`  
  無い場合は、`.addItem('請求フラグ', 'applyBillingFlagToSelection')` の次の行に上記を追加して保存する。その後、スプレッドシートを再読み込みする。

### 原因3：実行時エラー（シート名・選択範囲など）

- 「Excel実績自動化フラグ が見つかりません」→ 同じスプレッドシート内に **Excel実績自動化フラグ** という名前のシートがあるか確認する
- 「セル範囲を選択してから実行してください」→ 実行前に、**利用者様名が入っているセル**を選択してからメニューを実行する
- 選択範囲が A 列だと「B列以降で指定してください」と出る → B 列以降のセルを選択する

---

## 注意事項

- 選択セルは **B列以降** にしてください（1列左のヘルパー名参照のため）
- 9列右まで列があるシートで実行してください
- 色は `#RRGGBB` 形式で比較しています（大文字・小文字は区別しません）
